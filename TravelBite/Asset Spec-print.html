Type hierarchy

4 families · 12 roles

Four typefaces carry the entire app, each doing one job. DM Serif Display is the headline voice — warm, slightly bookish, used for dish names, screen titles and oversized numerals. Inter handles all running UI text and body copy. JetBrains Mono is the “stamped” voice for eyebrows, metadata, technical labels and anything that wants the authority of a passport entry. Caveat is the script note — sparingly used to give a recipe or postcard the hand-written feel of a travel journal.

Pairing rules

Style
  • Never set DM Serif under 13 px — the slabs collapse. Use Inter for small.
  • Always pair an eyebrow (mono) with a serif headline below. The contrast is the look.
  • One Caveat callout per screen, max. Script is precious; over-use = greeting card.
  • Numerals in stat tiles are DM Serif (not Inter) — the editorial feel.
  • Buttons are Inter 600, never serif. Save the serif for nouns, not verbs.
  • Mono tracking is always positive (+8% to +18%). It widens the rhythm so the labels feel stamped.
  • Color: serif = ink, body = ink-2, mono labels = ink-3, script = terracotta. Don’t cross-mix.

Color hierarchy

3 tiers · light + dark

Every color in the app belongs to one of three tiers: backgrounds (the surfaces things sit on), borders & strokes (the rules and dividers between things), and text (the ink itself, in three weights of emphasis). Accents — terracotta, turmeric, sage, plum — are summoned sparingly on top of these tiers and are documented in §03–§07.

2.1   Backgrounds  · 4 SURFACES

Surfaces step from page (the bottom-most paper) up to elevated (cards, modals, sheets). Higher tiers are slightly darker in light mode, slightly lighter in dark mode — the page should always feel like the deepest layer.

  Token Light Dark Role Where

--paper-2 #ece0c8 #241910 Inset surface — sits inside cards as a subtle well. Search wells, progress troughs, inactive chip backs.
--paper-3 #e3d3b3 #2f2317 Raised surface — cards, sheets, popovers, the cuisine wheel. All .card blocks, modals, bottom sheets.
--ink (inverted) #2a1d12 #f3e9d7 Inverted surface — high-contrast “stamp” backgrounds. Primary buttons, the dark hero band on Today’s Card.

2.3   Text  · 3 EMPHASIS LEVELS + ACCENTS

Three weights of ink, descending in importance. Ink is for headlines and the things you need to read first; ink-2 for body and supporting copy; ink-3 for labels, eyebrows, meta. Accent text is reserved for the script voice and a small set of UI states.

  Token Light Dark Role Where

Do

  • Stack tiers: page (paper) → inset (paper-2) → raised (paper-3). Don't skip.
  • Pair text + surface: ink on paper, ink-2 on paper-3 — never ink-3 on paper-3 (too low contrast).
  • Use dashed rule between list rows; solid rule for card outlines.
  • Test in dark: every token has a dark twin. If it doesn't, it isn't a token.

Don't

  • No new neutrals. If you want a new gray, you almost certainly want ink-2 or ink-3.
  • No accent text on accent surface. Terracotta text on a terracotta button = invisible.
  • No 1px gray hairlines below 0.2 alpha — they vanish on dark mode.
  • No raw hex in components. Always reference a token; the dark theme depends on it.

When to use which format

quick reference
.svg
vector · scalable

Spice discs, stamps, badges, all UI icons, source-app marks. Anything geometric.
Pros: tiny, retina-perfect, recolorable via currentColor.

.png-32
raster · alpha

When SVG isn't possible (textured stamps, distressed badges with raster effects). Always 3× device pixels; sRGB, no ICC profile.

.jpg
raster · photographic

All dish photography. q80–85, progressive, sRGB. No alpha needed — these always sit on solid card backgrounds.

.webp
raster · modern

Optional smaller-payload twin of every JPG. Serve via <picture> with JPG fallback. ~30% smaller at same quality.

★ Production notes

  • Density: all raster assets exported at 3× for Android xxhdpi. Generate 1× and 2× variants for older devices, or rely on a single 3× and let Android downscale.
  • Color profile: sRGB only. Strip ICC profiles to keep file size down; don't ship Display-P3 or Adobe RGB — they render dimmer on Android.
  • Transparency: stamps and badges must be transparent PNG-32 — they overlay rotated on photos. JPG would baked-in the white background.
  • Naming convention: dish-shakshuka@3x.jpg, spice-cumin-disc.svg, stamp-tunis.png, badge-rank-3.svg, source-tiktok.svg.
  • Trademarked marks: use only the official brand assets for TikTok, Instagram, Pinterest, YouTube. Do not recolor.
  • Dark-mode: all photos render against dark paper too — make sure subjects are well-lit so they don't disappear. Add a 1px rgba(255,255,255,0.06) inner stroke on `.food-ph` if needed.
  • Spice discs have a built-in printed-paper noise overlay applied via CSS — supply flat color circles, no texture.
